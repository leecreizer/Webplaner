# 몸통 이동 시 부착 상품 실시간 동반 이동 — 설계

날짜: 2026-07-07 · 상태: 승인됨(A안)

## 문제

- 도어(`parentId`)는 store 커밋 후에만 따라감 → 몸체 드래그/기즈모 이동 **중** 제자리에 남았다가 점프.
- 스태킹·내부 배치 상품은 관계가 저장되지 않아(순수 기하) 몸통 이동 후에도 **아예 따라가지 않음**.

## 결정 (A안: 드래그 시작 시 기하 판정 캡처)

스키마 변경 없이, 드래그 시작 순간 "몸통에 포함된 상품"을 계산해 동반 그룹으로 잡는다.

### 동반(follower) 판정 — `collectFollowers(body, placed)`

BFS(올린 것 위에 또 올린 것 연쇄 포함):

1. `parentId === host.id` 인 도어.
2. 기하 포함: 발자국 중심이 host 발자국 내부(±1cm) **그리고** 바닥높이(lift)가 host 바닥보다 위 ~ host 윗면 +5cm 이내 → 내부 수납·윗면 스태킹 모두 포함. (host보다 바닥이 낮은 상품은 내가 올라탄 받침이므로 제외)

### 적용 지점

| 경로 | 드래그 중 | 놓을 때 |
|---|---|---|
| 몸체 직접 드래그(`startBodyDrag`) | follower live Group에 몸통 델타(Δx,Δz,Δy) 그대로 적용 | 비도어 follower를 `store.update(x,z,lift)` 일괄 커밋. 도어는 몸통 커밋 → 기존 정렬 효과가 재배치 |
| 단일 선택 기즈모 | onMouseDown에 캡처, translate/rotate 델타를 follower에 live 적용 (회전은 몸통 중심 기준 오프셋 회전 + ry 가산) | 동일 커밋 (회전 포함) |

- 스냅·스태킹 대상(`others`)에서 follower 제외 — 몸통이 자기 자식에 스냅/올라타는 자가참조 방지.
- 다중 선택 기즈모는 이미 매 change마다 store 커밋(도어 정렬 효과 작동)이므로 변경 없음.

## 비범위

- 관계 영속화(`attachedTo`) 없음 — 매 드래그 시작 시 재판정.
- 리사이즈 핸들의 follower 보정 없음(현행 유지).
